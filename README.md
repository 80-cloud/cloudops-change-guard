# CloudOps Change Guard

AWS環境へのIaC変更（CloudFormation / Terraform）を**安全に統制する**変更管理アプリ。
`apply` の成功で終わらせず、**変更前の危険検知・リスクに応じた承認・変更後の正常性確認・失敗時の復旧余地**までを一貫して扱う。

**ライブデモ**：https://cloudops-guard.duckdns.org/ （登録不要・デモ管理者アカウントで、リスク判定〜承認〜完了ゲートまで操作できます）

> 設計の中心：「AWS環境の変更は、applyが成功すれば終わりではない。変更前に危険を見つけ、必要な承認を得て、変更後に正常性を確認し、失敗時にも復旧できる状態を残す。」

## 何が違うか

「変更管理」や「`plan` の承認」だけを行う仕組みは多いが、本アプリは **`apply` が通ったかではなく、安全に変わり・正常だと確認でき・失敗しても戻せる状態か**までを 1 つのライフサイクルで扱う点が異なる。

| よくあるやり方 | 本アプリの違い |
|---|---|
| フォーム記入ベースの変更申請 | 実際の IaC 差分を「リソース型 × アクション」に正規化して**リスクを機械判定**（記入内容でなく差分を見る） |
| `plan` の承認・ポリシーチェックで完了 | **`apply` の成否とサービス正常性を分離**し、両方そろうまで「完了」にしない**完了ゲート** |
| 一律の承認フロー | **リスク階層に応じて承認段数が変わる fail-closed**。条件を満たさない状態遷移は拒否（409） |
| 文字列一致でのリスク検出 | **データ駆動のルール（JSON）**で判定し、ルールを足すだけで拡張できる |

## 技術スタック

Java / Spring Boot ・ React / TypeScript ・ PostgreSQL ・ JWT ・ RBAC ・ Docker Compose。
AWS連携は抽象化インターフェース越し（MVPはモック、将来 CloudFormation / Terraform / AWS SDK）。

## ロール

| ロール | できること |
|---|---|
| REQUESTER | 変更申請の作成・自分の申請の閲覧/編集（承認済みは編集不可） |
| REVIEWER | レビュー・承認・却下・差し戻し |
| OPERATOR | 承認済み変更の実行・実施前後チェック記録・ロールバック記録 |
| ADMIN | 全申請閲覧・ポリシー管理・監査ログ閲覧 |

認可は**必ずバックエンドで検証**する。フロントの表示制御は補助に過ぎない。

## 想定される技術的リスク

| ID | リスク | 影響 | 緩和策 |
|---|---|---|---|
| TR-01 | リスク判定が文字列一致頼みで誤検知/見逃し | 危険変更を見逃す | 差分正規化＋構造化項目併用。ルール単体テスト・フィクスチャ回帰 |
| TR-02 | フロントのみの認可で抜ける（IDOR） | 他人の申請操作 | サーバー認可＋所有者検証。認可マトリクステスト |
| TR-03 | 状態遷移の不正・競合 | データ不整合 | ステートマシン＋楽観ロック。不正遷移を全件テスト |
| TR-04 | 監査ログ改ざん | 追跡性喪失 | 追記専用・更新/削除APIを設けない |
| TR-05 | IaC種別差（TF/CFN）でパーサ分岐が散らかる | 保守困難 | Port/Adapter＋正規化モデルで吸収 |
| TR-06 | AWS連携を密結合し将来差替不能 | 拡張阻害 | Port/Adapter。SDKをService層に漏らさない |
| TR-07 | 「IaC成功＝完了」と誤認 | 異常を見落とす | IaC適用成功とサービス正常性を別管理し COMPLETED 条件で区別 |
| TR-08 | 過剰設計・未使用コード | 複雑化 | MVPスコープ厳守・既存命名踏襲・不要な汎用化禁止 |

## 設計上の原則（禁止事項の機械的担保）

- 認可をフロントだけで済ませない（サーバー強制・IDOR対策）。
- リスク判定を単なる文字列一致で終わらせない（差分正規化＋構造化項目）。
- BLOCKポリシー違反を警告だけで通過させない（遷移をサーバーで拒否）。
- ステータスを自由に直接変更できる設計にしない（ステートマシン経由のみ）。
- 監査ログを更新・削除可能にしない（追記専用）。
- production 変更と development 変更を同じ統制にしない（環境×リスクで承認段数可変）。
- 大量の未使用コードや不要な抽象化を作らない（MVPスコープ厳守・既存命名踏襲）。
