# ADR-0005: ランタイム/フレームワークのバージョン方針（Java 25 と Spring Boot）

- ステータス: 採用
- 日付: 2026-07-01

## 背景
言語ランタイムは Java 25（LTS）を採用している。
一方でアプリケーションフレームワークは Spring Boot 3.5 系を使用しており、
この系列の OSS 保守期間は終盤に入っている。
また Spring Boot が Java 25 を公式の対応対象として明記するのは 4.0 以降である。
つまり「Java 25 × Boot 3.5」は動作はするが、フレームワーク側の公式対応表の内側ではない。

## 決定
当面は Spring Boot 3.5 系を維持し、Java 25 は Gradle の toolchain でコンパイル・実行する。
Spring Boot 4.x への移行は、基盤となる Spring Framework 7 系の破壊的変更を伴うため、
影響範囲を評価したうえで、まとまった作業として段階的に実施する。
現時点では機能要件を満たしており、移行を急ぐ差し迫った理由はないため優先度を下げる。

## 検討した代替案
- ただちに Boot 4.x へ移行する: 破壊的変更が広範で、安定して動作している構成を
  大きく変えるコストと不具合リスクが高い。
- Java を Boot 3.5 の公式対応内のバージョンへ下げる: Java 25（LTS）採用の方針に反する。

## 結果
- バージョンの整合状況と移行判断を明文化し、依存更新の際に参照できる。
- 保守終盤の期間は依存関係の脆弱性監視で補い、移行は計画作業として扱う。
