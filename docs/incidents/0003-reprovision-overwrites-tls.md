# 0003: 設定再生成スクリプトが TLS 設定を上書きして https が停止

- 日付: 2026-07-09
- 区分: インフラ / デプロイ手順（副作用の見落とし）

## 事象
https 化の後、アプリの環境設定を作り直すためにプロビジョニングスクリプトを再実行したところ、
https が応答しなくなった（443 が閉じ、平文 80 のみに戻った）。

## 原因
プロビジョニングスクリプトは Web サーバー設定を毎回「平文(80)版」で書き直す。
TLS 有効化スクリプトが入れた「TLS(443)版」設定を、あとから実行したプロビジョニングが上書きした。
環境設定の再生成が目的でも、同スクリプトが Web サーバー設定も一緒に初期化する副作用があった。

## 対策
TLS 有効化スクリプトを再実行して TLS 設定を再適用した（証明書は既存を再利用・冪等）。

## 再発防止（仕組みへの昇格）
- デプロイ手順の順序を固定：設定値の更新 → プロビジョニング（環境設定の再生成）→ TLS 有効化（必ず最後）。
- 手順書に「プロビジョニング再実行は Web サーバー設定を初期化するため、そのあと必ず TLS を再適用する」
  を明記。
- 将来は「環境設定の生成」と「Web サーバー設定の設置」を分離し、既存の TLS 設定を検知したら保護する。
