# 0004: 無料枠プランの制約で apply が失敗（バックアップ保持・EBS 最小サイズ）

- 日付: 2026-07-09
- 区分: インフラ / コスト・無料枠（既定値の不整合）

## 事象
実 apply で DB 作成と EC2 作成が失敗した。
DB は「バックアップ保持期間が無料枠の上限を超える」、EC2 は「ルートボリュームがイメージの
スナップショット(30GB)より小さい(20GB)」というエラー。

## 原因
新しい無料枠プランには従来と異なる制約がある（バックアップ保持の上限、追加機能の課金）。
また最新の OS イメージはルートスナップショットが 30GB に増えており、ボリュームは 30GB 以上が必要。
雛形の既定値（バックアップ保持 7 日・ボリューム 20GB・監視機能 有効）が新しい無料枠プランと合わなかった。

## 対策
バックアップ保持を無効化、ルートボリュームを 30GB、コスト増につながる監視機能を無効化して
無料枠の範囲に収めた（意図的な逸脱は設定ファイルに理由コメントを残す）。

## 再発防止（仕組みへの昇格）
- 雛形の既定値を「無料枠プランで通る値」にする（バックアップ保持・ボリューム最小・追加機能）。
- デプロイ前チェックに「無料枠プラン特有の制約（バックアップ保持上限・EBS 最小サイズ・追加機能の課金）」
  を 1 項目追加。
- 最新イメージ参照はスナップショットサイズが増えることがあるため、ボリューム下限を余裕を持って設定。
