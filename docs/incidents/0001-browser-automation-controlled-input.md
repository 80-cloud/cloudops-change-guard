# 0001: ブラウザ自動操作で制御された入力欄に値が反映されない

- 日付: 2026-06-29
- 区分: テスト / 画面確認（製品コードの不具合ではない）

## 事象
画面の動作確認で、ブラウザ自動操作から入力欄（複数行テキスト・選択リスト・日時）へ
値を入れても、送信時に空として扱われることがある。

## 原因
これらの入力欄は画面側で値を管理する制御コンポーネント。要素の value を直接書き換えるだけでは
画面側の状態更新イベントが発火せず、表示と内部状態がずれるため。

## 対策
値を設定したあと、対象要素へ input / change イベントを明示的に発火させ、画面側へ変更を伝える
（プロパティの setter 経由で値を入れ、続けてイベントを送る）。

## 再発防止（仕組みへの昇格）
- 表示部品はできるだけコンポーネント単体テストで検証し、手動・自動操作への依存を下げる。
- 画面確認の手順に「制御入力欄は値設定とイベント発火をセットで行う」を1項目として明記する。
