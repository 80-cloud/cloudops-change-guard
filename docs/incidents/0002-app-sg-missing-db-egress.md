# 0002: アプリのセキュリティグループに DB への外向き許可がなく起動失敗

- 日付: 2026-07-09
- 区分: インフラ / ネットワーク（Terraform 設定漏れ）

## 事象
公開デプロイ後、アプリが起動時に DB 接続タイムアウトでクラッシュし、再起動を繰り返した
（health が 502、DB 接続で接続タイムアウト）。

## 原因
DB 側セキュリティグループの受信（ingress）はアプリのセキュリティグループから 5432 を許可していたが、
アプリ側セキュリティグループの送信（egress）が 80/443 のみで、DB(5432) への外向きが無かった。
セキュリティグループはステートフルだが ingress と egress は独立しており、送信を最小化した際に
DB 向けの送信許可を足し忘れた。パケットは破棄されるため「拒否」ではなく「タイムアウト」になる。

## 対策
アプリのセキュリティグループに「DB のセキュリティグループ宛て 5432 の egress」を追加した。

## 再発防止（仕組みへの昇格）
- private なバックエンド（DB 等）への ingress を絞ったら、呼び出す側の egress も対で用意する
  （ingress/egress の対称性）。
- デプロイ前チェックに「アプリ SG から接続する各バックエンドへの egress が存在するか」を 1 項目追加。
- 静的解析（tflint / tfsec）では検知できないため、チェックリストで担保する。
