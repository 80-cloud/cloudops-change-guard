# 機能一覧（CloudOps Change Guard）

> 優先度は MoSCoW。Phase は実装順（[../README.md] 参照）。

| ID | 機能 | 優先度 | Phase | 受入条件 |
|---|---|---|---|---|
| F-AUTH-01 | JWT 認証（login/refresh/logout/me） | MUST | 2 | — |
| F-AUTH-02 | RBAC 認可（REQUESTER/REVIEWER/OPERATOR/ADMIN）をサーバー強制 | MUST | 2 | A-1, A-2 |
| F-CR-01 | 変更申請の作成・閲覧・編集・取消 | MUST | 2 | A-3 |
| F-CR-02 | ステータス状態遷移（12状態・ステートマシン） | MUST | 2 | A-9 |
| F-CR-03 | 一覧フィルタ（環境/ステータス/リスク/申請者）＋ページネーション | MUST | 2 | — |
| F-AUDIT-01 | 監査ログ記録（12操作種別・追記専用・改ざん不可） | MUST | 2 | A-8 |
| F-RISK-01 | リスク自動判定（差分正規化＋13ルール・説明文付き） | MUST | 3 | A-4, A-5 |
| F-POLICY-01 | ポリシーエンジン（BLOCK/追加承認/理由必須/メンテ時間/二重承認） | MUST | 3 | A-4, A-5, A-6 |
| F-APPROVE-01 | 承認フロー（環境×リスクで段数可変・自己承認禁止） | MUST | 3 | A-2, A-6 |
| F-PRECHK-01 | 実施前チェックリスト（本番は必須完了が実行開始の前提） | MUST | 4 | A-7 |
| F-HEALTH-01 | 実施後ヘルスチェック（IaC適用成功とサービス正常性を区別） | MUST | 4 | A-10 |
| F-DASH-01 | ダッシュボード（件数/高リスク/承認待ち/実施予定/最近の監査） | MUST | 4 | — |
| F-CMT-01 | コメント機能 | SHOULD | 3-4 | — |
| F-POLICY-02 | ポリシー一覧表示（ADMIN） | MUST | 3 | — |
| F-POLICY-03 | ポリシー GUI フル CRUD | MAY | 将来 | — |
| F-NOTIFY-01 | 外部通知（Slack/メール） | MAY | 将来 | — |
| F-AWS-01 | 実 AWS API / 実 apply 連携 | MAY | 将来 | — |
